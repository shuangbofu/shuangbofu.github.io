改变的第一天

Jun 03 2019

    今天是2019年06月03日，从早起开始都感觉特别的谨慎。也不知道为什么用谨慎这个词，就感觉每一刻每个呼吸都格外的小心，生怕又变得浮躁变得随便。第一天总是那么得容易坚持，不知道是不是昨天晚上睡觉前的冥想的缘故，这一整天我都处于和前几天完全不同的另外一种状态，呼吸缓慢不急不躁。

    早起带来的第一个好处呢自然是能吃早饭，舒舒服服的喝了一碗粥。等公交车的时候也比以往更加的耐心，按照计划在车上看《自控力》。

    今天读的内容总结一下大致有以下几点：

  1. 要认识自己，试着关注自己的内心世界，需要时刻认识到自己在做些什么。比如在面对拖延、冲动、诱惑的时候能够意识到自己的行为，在有这些冲动的行为的苗头前阻止自己。
  2. 通过训练自己的大脑提高意志力，一个很有效的方式是冥想。冥想的时候只需要把注意力集中在“呼”和“吸”上，而且完全不需要在乎是否会有其他杂乱的想法打断冥想，只需要每天能够坚持10-15分钟，甚至5分钟都可行，重在坚持。
  3. “三思而后行”也是一种人类的天性，提高自控力的一个方式是可以提高自身的心率变异度，深呼吸是一个很有效的方法。面对诱惑前深呼吸。
  4. 训练身心，锻炼和充足的睡眠很重要。

    中午很顺利的看了小猪佩奇英文版，学习英语也没有特别关注，重点全在剧情上了，虽然剧情有点幼稚不过依然很有趣。

    今天工作上简单改了下上周留下的项目，主要还是自己看rust文档。同时还看了下rust的web框架actix-web，其中有个很重要的概念，就重点了解了下Actor并发模型，看了不少的文档和博客，大概了解到actor模式当前流行的多线程并发中的内存共享不同，进一步抽象隐藏了线程这个概念。总结一下大致有以下几点：

  1. actor模型不需要消息共享，没有任何锁。
  2. 每个actor都是单线程串行处理消息，同一时刻只有一个actor处理消息
  3. 通过分发的方式，多个actor可以进行分工同时进行不同的工作
  4. 自身内部维护了一个状态，actor间一切都是通过消息传递，消息不可变，外部无法对actor直接进行调用，每个actor都对应一个mailbox队列负责缓存消息。
  5. 因为是以异步的形式，而且轻量级，可以将任务细分为多个细小的事件
  6. 使用Actor模型多个组件间维护一致性会比较困难（比如银行转账）

    被发现我在划水之后，又被安排了一个任务。公司要做的数据可视化，使用的是开源的框架davinci，在此基础上进行修改。现在的问题是，有个功能需要截屏获取图片的操作，但是因为工程前后端分离，所以在使用PhantomJS的时候填写了链接但是无法访问前端链接导致无法截屏，即使可以访问也需要登录依然无法截屏。
